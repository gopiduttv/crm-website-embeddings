## üéØ How It Works

### Before (Field-by-Field)
```
User fills email ‚Üí Event #1: { fieldName: "email", fieldValue: "..." }
User fills name  ‚Üí Event #2: { fieldName: "name", fieldValue: "..." }
User fills phone ‚Üí Event #3: { fieldName: "phone", fieldValue: "..." }
User closes tab  ‚Üí Event #4: { fields: { email, name, phone } }

Result: 4 events total
```

### After (Batch Capture)
```
User fills email ‚Üí Stored in memory (no event)
User fills name  ‚Üí Stored in memory (no event)
User fills phone ‚Üí Stored in memory (no event)
3 seconds pass   ‚Üí Event #1: { fields: { email, name, phone } }

Result: 1 event total (75% reduction!)
```

## üöÄ Key Features

### ‚úÖ **Debounced Capture**
- Waits X seconds after last field interaction
- Default: 5000ms (5 seconds)
- Demo: 3000ms (3 seconds)
- Timer resets on each new field

### ‚úÖ **Tab Switch Capture**
- Triggers immediately when user switches tabs
- Uses `visibilitychange` event
- No waiting for debounce
- Ensures data capture before user leaves

### ‚úÖ **Page Close Capture**
- Triggers when user closes tab/window
- Uses `beforeunload` event
- Final safety net for data capture

### ‚úÖ **Duplicate Prevention**
- Tracks `lastSent` timestamp for each form
- Skips if already sent within 1 second
- Prevents multiple events from different triggers

### ‚úÖ **Multi-Form Support**
- Tracks each form independently
- Separate state for each form ID
- No cross-contamination

### ‚úÖ **Configurable Thresholds**
- `minFieldsForCapture` - Minimum fields to send
- Can require at least 2-3 fields for partial leads
- Default: 1 (send even with one field)

## üìù Event Structure

**Batched Event**:
```json
{
  "type": "form_interaction",
  "apiKey": "sk_test_demo123",
  "timestamp": "2025-10-22T16:45:00.000Z",
  "form": {
    "formId": "contact-form",
    "trigger": "batch_capture",
    "fields": {
      "email": "john@example.com",
      "name": "John Doe",
      "phone": "+1-555-1234",
      "company": "Acme Corp"
    },
    "fieldCount": 4,
    "formProgress": {
      "completedFields": ["email", "name", "phone", "company"],
      "totalFields": 5,

    }
  },
  "page": {
    "path": "/demo",
    "title": "YourCRM - Demo Page",
    "search": "",
    "hash": ""
  }
}
```

**Key Differences from Old Format**:
- ‚úÖ `trigger: "batch_capture"` (not "blur")
- ‚úÖ `fields: {}` object (not single `fieldName`/`fieldValue`)
- ‚úÖ `fieldCount` - number of fields captured
- ‚úÖ All fields in one event (not multiple events)
